# AI Implementation Guidelines

このドキュメントは、AIが本プロジェクトにおいて過剰な実装（不要な装飾や複雑化）を行わないための行動規範です。

## 1. YAGNI (You Aren't Gonna Need It) & KISS (Keep It Simple, Stupid)
- ユーザーに明確に要求されていない機能や装飾を**絶対に独自解釈で追加してはならない**。
- 「プレミアム感」や「リッチなUI」を目指して、勝手なホバーエフェクト（`hover:scale`, `hover:-translate-y-1`等）、アニメーション（`transition-all`等）、グラデーション、シャドウエフェクトを付け足さない。
- 頼まれていないブレイクポイント対応（`sm:`, `md:`, `lg:`などの大量付与）による複雑なレスポンシブ実装を行わない。

## 2. 最もシンプルで直接的なコードを書く
- 既存のシンプルで意図が明確なコード（例えば `<img>` タグの羅列など）を、勝手に `Array.map` や過剰なコンポーネント分割を用いて抽象化しない。
- 冗長であっても、コードの意図が明確で後から誰でも保守しやすい形を優先すること。

## 3. デザインへの忠実な再現を最優先する
- デザインカンプ（画像など）が提供された場合は、目に見える要素をすべて配置する（例えば下段のSVGなど）ことを、見た目の装飾よりも優先する。
- 提供されたデザイン画像から「さらに良くしよう」と推測して手を加えることは厳重に禁じる。

## 4. セルフチェック
実装案を提出する前に以下の問いに答えること：
1. 「ユーザーが指示した文言以上のコードを足していないか？」
2. 「このスタイリングは本当にデザイン指示に存在したか？」
3. 「余計な抽象化によって、逆にコードの可読性を落としていないか？」
